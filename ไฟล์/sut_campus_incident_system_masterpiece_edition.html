3. Firebase Deep Dive (20-25%)

เจาะลึก Schema, Authentication Flow, Realtime Sync และ Push Notification (FCM)

Database Schema & Read/Write Patterns

📁 users/
uid // Document ID == Auth UID
studentId: String
role: String // "user", "dispatcher", "responder"
fcmToken: String // For Push Notification
department: String // If role == responder
📁 incidents/
incidentId // Auto ID
title: String
type: String // medical, security, accident
priority: String // HIGH, LOW
status: String // NEW, IN_PROGRESS, RESOLVED
timelineStatus: String
location: GeoPoint // Lat/Lng
createdBy: String // User UID
assignedResponder: String? // Nullable UID
createdAt: Timestamp
updatedAt: Timestamp

↳ 📁 messages/{messageId} (Subcollection)
↳ 📁 logs/{logId} (Subcollection)
เก็บ Audit log ทุกครั้งที่อัปเดตเคส ป้องกันการลบข้อมูล
Read/Write Patterns:
- Create: .collection('incidents').add()
- Update Status: ใช้ runTransaction() เพื่อหลีกเลี่ยง Race Condition หาก Dispatcher กด Assign พร้อมกัน
- Read: .snapshots() สำหรับ Dashboard
Firestore Indexes (Compound Queries):
ระบบต้องสร้าง Index ใน Firebase Console สำหรับคำสั่ง:
where('status', isNotEqualTo: 'RESOLVED').orderBy('createdAt', descending: true)
*หากไม่ทำ Index หน้า Dashboard จะแครช (Failed precondition)*

Realtime Synchronization Flow

ความลับของระบบที่ไม่ต้องมีปุ่ม Refresh คือการทำงานร่วมกันระหว่าง Firestore และ Riverpod State Management

1. Firestore Snapshot Listener (Data Layer)
FirebaseFirestore.instance.collection('incidents').snapshots()
// เปิดท่อ WebSocket ทิ้งไว้รอรับข้อมูลที่เปลี่ยน (Delta changes)
2. StreamProvider (Service Layer)
final incidentsProvider = StreamProvider((ref) => repository.getIncidentsStream());
// Riverpod นำ Stream มาแปลงเป็น State ที่ UI อ่านได้ง่าย
3. UI Rebuild (Presentation Layer)
ref.watch(incidentsProvider).when(
  data: (incidents) => ListView.builder(...),
  loading: () => CircularProgressIndicator(),
);
// UI วาดใหม่เฉพาะส่วนที่เปลี่ยนอัตโนมัติ ภายในเสี้ยววินาที

🔑 Auth Flow & Hack

Login Flow: 1. Input Student ID (`B64...`) + Password
2. Transform: แปลงเป็น B64...@campus.local (Hack เพื่อหลบข้อจำกัด Email ของ Firebase)
3. signInWithEmailAndPassword
4. Fetch Role จาก Firestore
5. Role-based Route ไปหน้าจอหลัก

🔔 FCM Delivery Lifecycle

Trigger: New Incident Created
↓ Cloud Function ตรวจพบ onCreate
↓ Query: users where role == 'dispatcher'
↓ admin.messaging().sendMulticast(tokens)
Trigger: Dispatch Assigned
↓ Cloud Function ตรวจพบ onUpdate
↓ ดึง FCM Token ของ assignedResponder
↓ admin.messaging().send(token)
↓ Responder Tap Noti → notification_router.dart เปิดหน้าเคส
