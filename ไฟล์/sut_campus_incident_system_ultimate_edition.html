Source Code Deep Dive

เจาะลึก lib/, แพ็กเกจที่ใช้, และโค้ดสำคัญระดับ Implementation

Firebase Auth บังคับใช้ Email แต่ มทส. ใช้รหัสนักศึกษา (เช่น B6412345) นี่คือทริคที่ใช้ใน auth_repository.dart:

// studentId "B6412345" → email "B6412345@campus.local" final studentEmail = '$normalizedId@campus.local'; UserCredential result = await _auth.signInWithEmailAndPassword( email: studentEmail, password: password, );

ใน role_redirect.dart หลังล็อกอิน ระบบจะไปดึง Field role จาก Firestore เพื่อนำทางอย่างปลอดภัย:

if (userRole == 'user') { return HomeScreen(); } else if (userRole == 'dispatcher') { return DispatcherScreen(); } else if (userRole == 'responder') { return ResponderDashboard(); }

ป้องกัน Dispatcher แย่งกันกดรับเคสเดียวกันในเสี้ยววินาที:

await _firestore.runTransaction((transaction) async { final snapshot = await transaction.get(incidentRef); if (snapshot.data()['status'] != 'NEW') { throw Exception("มีเจ้าหน้าที่คนอื่นรับเคสนี้ไปแล้ว!"); } transaction.update(incidentRef, {'status': 'IN_PROGRESS'}); });

📱 Flutter Mobile Tier

ในฝั่ง Mobile (สำหรับ Reporter และ Responder) โค้ดถูกจัดระเบียบแบบ Feature-First ภายในโฟลเดอร์ lib/features/

lib/features/
├── auth/ // Login & Role Redirect
├── incident/ // Forms & Repositories
├── responder/ // GPS Tracking Module
└── notification/ // FCM Receiver

มีการใช้ geolocator ดึงค่า High Accuracy GPS ส่งขึ้นคลาวด์ทุกๆ วินาทีสำหรับฟีเจอร์ F6 (Real-time tracking)

💻 Web Dashboard Tier

หน้าจอ DispatcherScreen และ AdminDashboard ถูกออกแบบมาเพื่อ Web โดยเฉพาะ จุดที่น่าสนใจคือการจัดการ Notification

ปัญหา: Web Service Worker จัดการ FCM ยุ่งยากมาก
ทางแก้จาก Source Code: สร้างไฟล์ web_notification_watcher.dart เป็น Wrapper Widget ที่คอย Listen `.snapshots()` ของ Firestore โดยตรง ถ้ายอดเอกสารเพิ่ม = เด้งแจ้งเตือนผ่าน UI ทันทีไม่ต้องพึ่ง FCM ของ Google

🗺️ Adaptive Map Architecture

ทำไมโปรเจกต์นี้ถึงใช้ Library แผนที่ถึง 2 ตัว (Google Maps & Leaflet)? คำตอบคือ Performance Trade-offs

  • Mobile (google_maps_flutter): ใช้ Native SDK ของเครื่อง ลื่นไหล เรนเดอร์ 3D ได้ดีเยี่ยม
  • Web (flutter_map): หากใช้ Google Maps บนเว็บ Flutter จะแปลงเป็น WebGL ซึ่งอาจมีปัญหากับคอมพิวเตอร์ศูนย์วิทยุเก่าๆ การหันมาใช้ Leaflet (Canvas ธรรมดา) ช่วยให้อ่านแผนที่ใหญ่ๆ ได้เสถียรกว่ามาก

🔑 Firebase Authentication

ระบบจัดการตัวตนของผู้ใช้งาน (Identity Management) ที่ถูกปรับแต่งเพื่อรองรับรหัสนักศึกษาของ มทส.

เทคนิคพิเศษ (Auth Hack): Firebase บังคับใช้ Email แต่ระบบแปลงรหัสนักศึกษา (เช่น B6412345) เป็น B6412345@campus.local ก่อนส่งเข้าระบบ เพื่อให้ล็อกอินด้วยรหัสนักศึกษาได้โดยตรง

🗄️ Cloud Firestore

ฐานข้อมูล NoSQL แบบ Real-time ที่เป็นหัวใจหลักในการ Sync ข้อมูลของระบบ 3-Tier นี้

  • Real-time Stream: ใช้ .snapshots() เพื่อดันข้อมูลใหม่ไปยังหน้าจอ Dispatcher ทันทีที่ผู้ใช้แจ้งเหตุ
  • Case Isolation: กฎ firestore.rules บล็อกการเข้าถึง Subcollection messages อย่างเข้มงวด ให้อ่านได้เฉพาะเจ้าของเคสและเจ้าหน้าที่ที่ได้รับมอบหมายเท่านั้น
